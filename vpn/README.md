# 🛡️ Собственный AmneziaWG VPN Сервер

**AmneziaWG** — это форк популярного протокола WireGuard, разработанный командой Amnezia VPN специально для работы в условиях жесткой интернет-цензуры. Он полностью совместим с классическим WireGuard, но защищен от блокировок по сигнатурам с помощью добавления мусорных пакетов в рукопожатие и изменения структуры заголовков.

В этом разделе описана простая установка AmneziaWG-сервера с веб-панелью управления **amnezia-wg-easy** для удобного создания пользователей и скачивания готовых конфигураций/QR-кодов.

---

## 📐 Архитектурный разбор параметров маскировки AmneziaWG

Стандартный протокол WireGuard имеет строго фиксированную структуру пакетов и заголовков, из-за чего легко идентифицируется системами DPI (ТСПУ) по сигнатурам. AmneziaWG решает эту проблему за счет рандомизации структуры трафика на этапе рукопожатия (handshake).

```
Классический WireGuard:
[Handshake Initiation] -> Фиксированный размер (148 байт) -> Узнаваемый заголовок (0x01)

AmneziaWG:
[Junk Packets] + [Initiation + Random Bytes] -> Случайный размер -> Измененный заголовок
```

### 1. Изменение длины пакетов: параметры Jc, Jmin, Jmax
При инициализации сессии классический WireGuard отправляет пакет `Handshake Initiation` строго определенного размера — 148 байт. DPI цензора мгновенно выявляет эту сигнатуру.
AmneziaWG маскирует размер пакетов с помощью следующих параметров:
*   **`Jc` (Junk count):** Количество мусорных пакетов, которые клиент отправляет на сервер перед началом реального рукопожатия (и которые сервер отправляет обратно). Обычно устанавливается значение от `3` до `5`.
*   **`Jmin` и `Jmax`:** Минимальный и максимальный размер мусора (рандомных байт) в байтах. Мусорные байты прикрепляются к телу пакетов рукопожатия, изменяя их размер со стандартного 148 до случайного числа (например, от `50` до `1000` байт).
*   Для DPI это выглядит как передача случайных неструктурированных UDP-данных разной длины, что лишает цензора возможности заблокировать сессию по фиксированному размеру кадра.

### 2. Изменение магических чисел заголовков: параметры H1-H4
Классический WireGuard использует фиксированные байты в заголовках пакетов для определения их типа:
*   `0x01` — Handshake Initiation
*   `0x02` — Handshake Response
*   `0x03` — Cookie Reply
*   `0x04` — Transport Data
DPI-системы проверяют первые байты каждого UDP-пакета на соответствие этим «магическим числам».
AmneziaWG подменяет эти константы на кастомные значения:
*   **`H1`–`H4`** — новые байты заголовков для каждого типа пакетов соответственно (задаются целыми числами от 1 до 255).
*   **`S1` и `S2`** — параметры дополнительного смещения байт в заголовках пакетов инициализации и ответа для смещения криптографического материала.

> [!IMPORTANT]
> Параметры маскировки (`Jc`, `Jmin`, `Jmax`, `S1`, `S2`, `H1`-`H4`) являются предопределенными константами шифрования. Они **должны строго совпадать** на сервере и у клиента. Если клиент попытается подключиться с несовпадающими заголовками или смещениями, сервер не сможет дешифровать рукопожатие, сочтет пакет нелегитимным шумом и проигнорирует его.

## 🚀 Установка сервера

### 1. Подготовка папки
Создайте рабочую директорию на вашем сервере VPS:
```bash
mkdir -p /opt/amnezia-wg && cd /opt/amnezia-wg
```
*(Скопируйте в эту папку [`docker-compose.yml`](./docker-compose.yml) из этого репозитория).*

### 2. Редактирование параметров
Откройте `docker-compose.yml` and настройте переменные:
*   `WG_HOST` — укажите публичный IP-адрес вашего VPS.
*   `PASSWORD` — укажите надежный пароль для входа в веб-интерфейс панели управления.

### 3. Открытие портов
В брандмауэре вашего хостинга откройте порты:
*   `51820 / UDP` — для VPN трафика.
*   `51821 / TCP` — для доступа к панели администратора (рекомендуется закрыть его или ограничить доступ только с вашего IP после настройки, либо использовать SSH-туннель).

### 4. Запуск сервера
Запустите Docker контейнер:
```bash
docker compose up -d
```
Служба автоматически поднимет туннель, настроит NAT-маршрутизацию на сервере и запустит веб-интерфейс.

## 🖥️ Создание пользователей и подключение

1. Откройте браузер и перейдите на страницу: `http://IP-адрес-вашего-сервера:51821`
2. Введите пароль, указанный в `docker-compose.yml`.
3. Нажмите кнопку **+ Client** (Создать клиента) и введите имя пользователя (например, `phone-stepan`).
4. Рядом с созданным пользователем появятся кнопки:
   *   **QR-код** — для быстрого скачивания на телефон.
   *   **Скачать конфиг (.conf)** — для загрузки на ПК или роутер.
   *   **Удалить/Деактивировать** — для управления доступом.

Конфигурационный файл, скачиваемый из этой панели, уже содержит все необходимые настройки обхода DPI (`Jc`, `Jmin`, `Jmax`, `S1`, `S2`, `H1-H4`), поэтому никаких дополнительных ручных правок делать не нужно.

## 📱 Клиенты для AmneziaWG

### 💻 Компьютеры (Windows, macOS, Linux)
1. Скачайте официальный клиент AmneziaWG:
   *   [Релизы на GitHub](https://github.com/amnezia-vpn/amneziawg-windows/releases) (для Windows)
   *   [Сайт Amnezia VPN](https://amnezia.org/ru/amneziawg) (все платформы)
2. Импортируйте скачанный `.conf` файл.
3. Нажмите кнопку **Подключить**.

### 📱 Смартфоны (Android, iOS)
1. Установите приложение **AmneziaWG** из Google Play или App Store.
2. Откройте приложение, нажмите **+** -> **Сканировать QR-код**.
3. Отсканируйте код из веб-интерфейса сервера и подключитесь.

### 🔌 Настройка на роутерах Keenetic
1. Скачайте `.conf` файл клиента из веб-интерфейса сервера.
2. Зайдите в веб-интерфейс Keenetic -> **Другие подключения** -> **Добавить подключение**.
3. Выберите тип: **AmneziaWG**.
4. Заполните поля из `.conf` файла (Приватный ключ, Адрес, DNS, публичный ключ пира, Endpoint и параметры обхода DPI: `Jc=4`, `Jmin=50`, `Jmax=1000`, `S1=15`, `S2=44`, `H1=1`, `H2=2`, `H3=3`, `H4=4`).
5. Сохраните подключение и назначьте его основным или настройте маршруты для конкретных домашних устройств.

## 🧰 Troubleshooting и отладка AmneziaWG

### 1. Ошибка «No handshake response» (Отсутствует ответ на рукопожатие)
Это самая распространенная ошибка клиентов WireGuard/AmneziaWG. Она означает, что клиент отправляет пакет инициализации, но не получает обратного ответа от сервера.
*   **Причина 1: Несовпадение параметров маскировки.** Проверьте, совпадают ли параметры `Jc, Jmin, Jmax, S1, S2, H1-H4` в конфигурационном файле клиента с параметрами, заданными на сервере. Если на сервере установлен стандартный AmneziaWG, а клиент пытается подключиться через классический WireGuard-клиент (без поддержки этих параметров), сервер проигнорирует запрос.
*   **Причина 2: Блокировка порта/IP цензором.** ТСПУ может полностью заблокировать UDP-порт сервера (по умолчанию `51820`), либо внести IP-адрес сервера в черный список. 
    *   *Как проверить:* Попробуйте сменить порт на сервере (например, настроить на порт `443/udp` или любой случайный порт) и обновить конфиг клиента.
*   **Причина 3: Блокировка брандмауэром (Firewall).** Убедитесь, что на сервере порт `51820/udp` открыт в `ufw` или правилах облачного провайдера.

### 2. Оптимизация MTU для мобильных сетей
Мобильные операторы связи часто используют инкапсуляцию трафика (например, GTP-туннели), что снижает эффективный размер кадра (MTU) в их сетях.
*   Если в конфиге клиента установлен стандартный `MTU = 1420` или `1500`, большие пакеты будут фрагментироваться. ТСПУ мобильных операторов часто отбрасывают фрагментированные UDP-пакеты, вызывая обрывы связи и падение скорости.
*   **Решение:** Уменьшите размер MTU в конфигурационном файле клиента до `1360`, `1280` или `1200` в секции `[Interface]`:
    ```ini
    MTU = 1280
    ```
    Это предотвратит фрагментацию и повысит стабильность соединения на мобильном интернете.
