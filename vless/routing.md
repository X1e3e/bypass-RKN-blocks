# 🔀 Маршрутизация трафика на клиентах VPN

Правильная настройка маршрутизации позволяет разделить ваш интернет-трафик: пускать заблокированные ресурсы (YouTube, Discord, Instagram) через прокси-туннель (PROXY), а доверенные российские сервисы (банки, Госуслуги, Кинопоиск) направлять напрямую (DIRECT). Это сохраняет высокую скорость соединения для локальных сайтов и предотвращает блокировку ваших аккаунтов банками из-за входа с зарубежных IP-адресов.

---

## 1. Базовые списки маршрутизации (Geosite и GeoIP)

Ядро Xray и большинство современных клиентов (NekoBox, Sing-box, Mihomo) используют предкомпилированные базы данных доменов и IP-адресов для фильтрации трафика:
*   **`geosite`** — база данных доменных имен, сгруппированная по категориям (например, `geosite:youtube`, `geosite:google`, `geosite:category-ru`).
*   **`geoip`** — база данных диапазонов IP-адресов по странам (например, `geoip:ru` для российских IP, `geoip:private` для локальных домашних подсетей).

---

## 2. Логика настройки правил (Сверху вниз)

Списки правил в клиентах всегда обрабатываются по принципу **«First Match» (Первое совпадение)** — трафик проверяется по правилам сверху вниз. Первое сработавшее правило определяет судьбу пакета, остальные правила игнорируются.

### Оптимальная структура правил в клиенте:

1.  **Блокировка рекламы и трекеров (Реклама -> BLOCK):**
    *   *Тип правила:* Домены
    *   *Значение:* `geosite:category-ads-all`
    *   *Действие:* **BLOCK** (или Blackhole)
2.  **Обход блокировок (Целевые ресурсы -> PROXY):**
    *   *Тип правила:* Домены
    *   *Значение:* `geosite:youtube,discord,instagram,twitter,facebook,google`
    *   *Действие:* **PROXY** (выбор вашего VLESS/Reality сервера)
3.  **Локальные российские сервисы (РФ ресурсы -> DIRECT):**
    *   *Тип правила:* Домены и IP-адреса
    *   *Значение:* `geosite:category-ru`, `geoip:ru`, `geoip:private`
    *   *Действие:* **DIRECT** (прямое подключение без VPN)
4.  **Финальное дефолтное правило (Все остальное):**
    *   **Вариант А (Режим «Обход блокировок»):** Направлять весь остальной трафик напрямую (**DIRECT**). Через VPN пойдут только явно указанные заблокированные сайты.
    *   **Вариант Б (Режим «Полный VPN»):** Направлять весь остальной трафик через прокси (**PROXY**). Весь ваш трафик шифруется, кроме российских сайтов.

## 3. Нюансы DNS-маршрутизации (Утечки DNS)

Чтобы маршрутизация по доменам работала корректно, ваш клиент должен правильно обрабатывать DNS-запросы.
*   **Проблема:** Если клиент сначала делает DNS-запрос через системный DNS вашего провайдера, провайдер увидит, какой сайт вы пытаетесь открыть, и заблокирует соединение на этапе DNS-запроса (или вернет подмененный IP-адрес заглушки РКН). Это называется **утечкой DNS (DNS Leak)**.
*   **Решение:**
    1. Настройте Xray в режиме перехвата DNS (снаффинг доменов `sniffing: true`). Клиент перехватит DNS-запрос до отправки в сеть, определит, к какому правилу относится домен, и перенаправит трафик в нужный туннель.
    2. Используйте защищенные DNS-серверы внутри Xray: например, `8.8.8.8` или `1.1.1.1` для прокси-трафика, и локальные DNS (например, `77.88.8.8` Яндекс DNS) для DIRECT трафика.

## 4. Продвинутая маршрутизация на роутерах (OpenWrt / Keenetic)

При настройке обхода блокировок на роутере для всей домашней сети используются два основных подхода маршрутизации:

### Подход А: Маршрутизация по спискам IP-адресов (через BGP / IPset / NFTset)
*   Роутер автоматически скачивает списки заблокированных IP-адресов (например, с сервиса Antizapret или Роскомсвобода).
*   При обращении к этим IP трафик заворачивается в VPN-интерфейс (sing-box/wireguard). Все остальные IP идут через основного провайдера.
*   *Плюсы:* Минимальная нагрузка на процессор роутера.
*   *Минусы:* Если заблокированный сайт часто меняет IP-адреса (как YouTube), списки могут не успевать обновляться.

### Подход Б: DNS-фильтрация (через dnsmasq + ipset / AdGuard Home)
*   При каждом DNS-запросе роутер проверяет, входит ли домен в список заблокированных (например, `youtube.com`).
*   Если входит, роутер резолвит IP-адрес через VPN-канал, временно добавляет полученный IP-адрес в динамическую таблицу маршрутизации (`ipset` или `nftables`) и направляет весь последующий трафик к этому IP через VPN.
*   *Плюсы:* Идеально работает для динамических сервисов и CDN.
*   *Минусы:* Требует больше оперативной памяти и настройки утилит dnsmasq / iptables на роутере.
