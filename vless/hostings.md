# 🏢 Хостинги для VPN: Выбор, Специфика и Методы Тестирования

Выбор качественного VPS-хостинга — важнейший шаг. Сервер в РФ (транзитный) должен обладать хорошей связностью с российскими провайдерами и лояльным отношением к трафику, а зарубежный (выходной) — чистыми IP-адресами, которые не вызывают постоянных капч в Google/Cloudflare, и стабильным каналом связи.

---

## 1. Анализ хостинг-провайдеров

На основе реального опыта администраторов и отзывов сообщества сформирован следующий перечень хостингов для создания VPN-инфраструктуры.

### 🇷🇺 Российские транзитные хостинги (RU Gateways)

*   **МТС Веб-Сервисы (MWS):**
    *   **Специфика:** Самый популярный вариант для обхода Белых Списков.
    *   **Нюанс с IP:** При заказе виртуальных машин (VMware Cloud) необходимо ловить IP-адреса из диапазонов мобильного пула МТС/Yota: `176.109.92.0/24`, `176.109.94.0/24`, `176.109.85.0/24` и `176.109.82.0/24`. Трафик к этим подсетям не фильтруется ТСПУ, так как они классифицируются как служебные сотовые сети.
*   **Aeza (локация РФ):**
    *   **Специфика:** Предоставляют мощные серверы на процессорах Ryzen 9 9950X (в пуле `83.147.255.0/24`) со скоростью портов до 25 Гбит/с.
    *   **Плюсы:** Удобная оплата картами РФ, встроенная базовая защита от DDoS, высокая производительность. Для маршрутизации заблокированных сайтов на самом сервере отлично подходит накатка `warp-native`.
*   **Selectel:**
    *   **Специфика:** Профессиональное облако корпоративного уровня.
    *   **Минусы:** Очень строгие лимиты на трафик. Перерасход свыше лимита (обычно 3 ТБ) тарифицируется по цене около 1.2 ₽/ГБ, что делает его дорогим для публичных VPN. Рекомендуется использовать только для небольших личных транзитных нод.
*   **Datacheap (`vps.datacheap.ru`):**
    *   **Специфика:** Бюджетный хостинг с серверами в Москве.
    *   **Плюсы:** Дешевые тарифы без жестких ограничений скорости порта при росте объемов трафика. Подходит для транзитных нод.
*   **VPSPay / Tube VDS:**
    *   **Предупреждение:** Имеют плохую репутацию в сообществе. Фиксируются сильные просадки скорости («зарезание полосы») при постоянной нагрузке, а также частые технические сбои из-за DDoS-атак на инфраструктуру провайдера.

### 🌐 Зарубежные хостинги (Выходные ноды)

*   **PQ Hosting / RuVDS (зарубежные локации):**
    *   **Плюсы:** Доступна оплата российскими картами, СБП и криптой. Широкий выбор локаций (Нидерланды, Германия, США, Турция).
*   **BuyVM:**
    *   **Плюсы:** Дешевые VPS с безлимитным трафиком на портах 1 Гбит/с. Возможность дешево докупить дисковые хранилища (Block Storage Slabs). Оплата криптовалютой.
*   **OVH / Hetzner:**
    *   **Плюсы:** Крупнейшие европейские ДЦ, высокое качество портов, стабильность.
    *   **Минусы:** Требуют строгой верификации аккаунтов (KYC с отправкой документов/селфи). Не принимают карты РФ. IP-адреса Hetzner часто находятся в черных списках РКН (блокируются целыми подсетями).

## 2. Методы тестирования и проверки серверов

Перед запуском VPN-сервера в эксплуатацию обязательно проведите диагностику сетевых параметров.

### 📶 1. Проверка скорости порта (Speedtest)
Вместо установки тяжелого официального клиента используйте легковесный Python-скрипт для измерения реальной скорости порта на сервере:
```bash
curl -s https://raw.githubusercontent.com/sivel/speedtest-cli/master/speedtest.py | python3 -
```
*Запустите тест несколько раз в часы пиковой нагрузки (с 20:00 до 23:00 по МСК), чтобы увидеть реальную пропускную способность канала.*

### 🛠️ 2. Трассировка маршрута и потеря пакетов (MTR по TCP)
Стандартный ICMP-пинг часто деприоритизируется сетевым оборудованием или блокируется ТСПУ. Для точной диагностики сетевого маршрута и поиска потерь пакетов используйте утилиту **MTR** в режиме TCP на порту `443`:
```bash
# Установка
apt update && apt install mtr -y

# Запуск TCP трассировки (отправка 50 пакетов на порт 443)
mtr -T -r -c 50 <IP_АДРЕС_ВАШЕГО_СЕРВЕРА> -P 443
```
*Обратите внимание на колонку Loss%: если на последних узлах перед сервером есть потери пакетов > 1-2%, Reality-соединение будет работать нестабильно.*

### 🔍 3. Проверка качества IP-адреса (IP Quality Score / Antifraud)
Если вы получили "грязный" IP-адрес с плохой историей (с него рассылали спам или проводили атаки), ваши пользователи будут постоянно сталкиваться с капчами в Cloudflare, Google, а также блокировками в стриминговых сервисах (Netflix).
*   **Как проверить:** Воспользуйтесь бесплатными чекерами репутации IP, например **[IPQualityScore](https://www.ipqualityscore.com/free-ip-lookup-proxy-vpn-test)** или **[Scamalytics](https://scamalytics.com/)**.
*   **Порог:** Показатель `Fraud Score` должен быть ниже **30**. Если он выше 50, рекомендуется сдать сервер обратно в течение тестового периода и запросить замену IP-адреса.

### 🚫 4. Проверка блокировки IP-адреса в РФ
Прежде чем настраивать Reality, проверьте, не заблокирован ли IP-адрес вашего зарубежного сервера системами РКН.
*   **Локальная проверка с домашнего ПК:**
    Попробуйте пинговать сервер по TCP с вашего домашнего компьютера (через терминал):
    ```bash
    # Для Windows (требуется tcping.exe):
    tcping <IP_СЕРВЕРА> 443
    
    # Для macOS / Linux:
    nc -zv -w 3 <IP_СЕРВЕРА> 443
    ```
*   **Использование публичных Looking Glass:** Используйте сервисы проверки доступности портов из разных точек РФ (например, `ping-admin.ru` или `host-tracker.com`), чтобы убедиться, что IP доступен со всех российских провайдеров.
