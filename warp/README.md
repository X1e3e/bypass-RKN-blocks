# 🌀 Cloudflare WARP + AmneziaWG

**Cloudflare WARP** — это бесплатный и безлимитный VPN от компании Cloudflare, предоставляющий высокую скорость и низкий пинг. Однако стандартное приложение WARP и протокол WireGuard заблокированы большинством провайдеров в РФ (блокируется характерный паттерн рукопожатия / handshake WireGuard).

**Решение:** Использовать **AmneziaWG** — форк WireGuard с защитой от DPI. Он добавляет случайный мусор (junk packets) в начало соединения и изменяет заголовки пакетов, маскируя трафик под случайный шум.

---

## 🛠️ Как это работает в России

Чтобы запустить WARP в России, нужно выполнить два шага:
1. Получить стандартную конфигурацию WireGuard для вашего аккаунта WARP.
2. Сконвертировать её в формат AmneziaWG, добавив параметры обхода DPI, и заменить заблокированный IP-адрес эндпоинта на рабочий.

---

## 🚀 Пошаговая инструкция

### Шаг 1. Генерация конфигурации WARP
Для генерации конфига воспользуйтесь готовыми инструментами:
*   **Скрипт в браузере (Google Colab):** Найдите в сети проекты по запросу "Warp-to-WireGuard generator" (например, от *Viacheslav* или аналогичные).
*   **Консольная утилита:** Используйте [wgcf](https://github.com/ViRb3/wgcf):
    ```bash
    wgcf register
    wgcf generate
    ```
    В результате вы получите файл `wgcf-profile.conf`, содержащий ваши приватные ключи (`PrivateKey`), адреса (`Address`) и стандартный заблокированный эндпоинт `engage.cloudflareclient.com:2408`.

---

### Шаг 2. Конвертация в AmneziaWG и настройка обхода DPI
Откройте полученный `.conf` файл в текстовом редакторе и приведите его к формату AmneziaWG.

#### 1. Добавьте параметры обхода в секцию `[Interface]`
Добавьте в секцию `[Interface]` следующие параметры:
```ini
Jc = 4
Jmin = 50
Jmax = 1000
S1 = 15
S2 = 44
H1 = 1
H2 = 2
H3 = 3
H4 = 4
```
*   `Jc` — количество мусорных пакетов при инициализации.
*   `Jmin` и `Jmax` — минимальный и максимальный размер мусора.
*   `S1`, `S2` — смещение размеров пакетов инициализации.
*   `H1`–`H4` — измененные байты заголовков для обхода сигнатурного детекта DPI.

#### 2. Замените заблокированный Endpoint
Стандартный домен `engage.cloudflareclient.com:2408` заблокирован ТСПУ. Вместо него нужно использовать IP-адреса Cloudflare. К сожалению, популярные IP (например, `162.159.192.1`) также могут быть частично заблокированы.

Для поиска работающих IP-адресов используйте сканеры:
*   **[WARP-IP-Scanner (GitHub)](https://github.com/bepass-org/warp-scanner):** Утилита сканирует диапазоны IP Cloudflare и находит те, которые не заблокированы вашим провайдером.
*   Пример рабочего эндпоинта (может меняться): `162.159.193.10:2408` (или порты `500`, `854`, `901`, `1701`, `4500`).

#### Пример итогового конфига `warp_amnezia.conf`:
```ini
[Interface]
PrivateKey = <ВАШ_ПРИВАТНЫЙ_КЛЮЧ>
Address = 172.16.0.2/32, 2606:4700:110:8f8a:e1f5:1234:5678:90ab/128
DNS = 1.1.1.1, 8.8.8.8
# Параметры AmneziaWG:
Jc = 4
Jmin = 50
Jmax = 1000
S1 = 15
S2 = 44
H1 = 1
H2 = 2
H3 = 3
H4 = 4

[Peer]
PublicKey = bmXOC+F1KF...
Endpoint = 162.159.193.5:1701
AllowedIPs = 0.0.0.0/0, ::/0
```

## 📱 Настройка на устройствах

### 💻 Windows / macOS / Linux
1. Скачайте официальный клиент AmneziaWG для вашей ОС с сайта [Amnezia.org](https://amnezia.org/ru/amneziawg) или [GitHub](https://github.com/amnezia-vpn/amneziawg-windows/releases).
2. Нажмите **Добавить туннель** (Add Tunnel) -> выберите ваш сконвертированный файл `warp_amnezia.conf`.
3. Нажмите **Подключить**.

### 📱 Android / iOS
1. Установите приложение **AmneziaWG** из Google Play или App Store (это официальный форк WireGuard-клиента, поддерживающий кастомные параметры).
2. Нажмите кнопку **+** -> **Импорт из файла или архива** и выберите ваш конфиг.
3. Включите соединение.

### 🔌 На роутерах Keenetic
Начиная с версии KeenOS 4.2, роутеры Keenetic поддерживают AmneziaWG "из коробки" в качестве стандартного VPN-подключения.
1. Зайдите в веб-интерфейс Keenetic -> **Другие подключения** -> **Добавить подключение**.
2. Выберите тип: **AmneziaWG**.
3. Заполните поля ключей, адресов и параметров `Jc, Jmin, Jmax, S1, S2, H1-H4` из вашего конфига.
4. Настройте приоритеты подключений (политики маршрутизации), чтобы пустить через WARP нужные устройства или только определенные домены.
