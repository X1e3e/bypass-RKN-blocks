# 🌀 Cloudflare WARP + AmneziaWG

**Cloudflare WARP** — это бесплатный и безлимитный VPN от компании Cloudflare, предоставляющий высокую скорость и низкий пинг. Однако стандартное приложение WARP и протокол WireGuard заблокированы большинством провайдеров в РФ (блокируется характерный паттерн рукопожатия / handshake WireGuard).

**Решение:** Использовать **AmneziaWG** — форк WireGuard с защитой от DPI. Он добавляет случайный мусор (junk packets) в начало соединения и изменяет заголовки пакетов, маскируя трафик под случайный шум.

---

## 📐 Architecture of Cloudflare WARP

Сеть Cloudflare WARP построена на глобальной инфраструктуре CDN Cloudflare и имеет ряд архитектурных особенностей:

### 1. Anycast IP маршрутизация
Cloudflare использует технологию **Anycast** для маршрутизации трафика клиентов:
*   Все пограничные дата-центры Cloudflare (PoP — Points of Presence) анонсируют одни и те же пулы IP-адресов эндпоинтов (например, подсеть `162.159.192.0/24`).
*   Когда клиент отправляет пакет на эндпоинт WARP, BGP таблицы маршрутизации автоматически направляют этот пакет в физически ближайший дата-центр Cloudflare.
*   Это обеспечивает минимальный RTT (время отклика) и высокую скорость загрузки, так как трафик распределяется по всей мировой сети.

### 2. Выделение внутренних адресов клиентам в сети WARP
При регистрации аккаунта WARP Cloudflare выделяет клиенту фиксированные внутренние IP-адреса внутри своего туннеля:
*   Обычно это один IPv4 адрес из диапазона `172.16.0.2/32` и один IPv6 адрес из подсети `2606:4700:110::/64`.
*   Эти адреса используются для маршрутизации трафика внутри виртуальной частной сети Cloudflare. На выходе из сети Cloudflare (при обращении к целевым сайтам) трафик маскируется под внешние IP-адреса дата-центров Cloudflare (NAT).

### 3. Механизм блокировки WARP со стороны ТСПУ
Стандартный протокол WireGuard (на котором работает оригинальное приложение WARP) блокируется ТСПУ по поведенческому паттерну:
1.  **Поиск фиксированной длины кадра:** Пакет инициализации рукопожатия (Handshake Initiation) классического WireGuard всегда имеет размер 148 байт.
2.  **Поиск заголовков:** Первые байты пакета содержат фиксированную структуру (например, `0x01 0x00 0x00 0x00` для пакета инициализации).
3.  ТСПУ провайдеров отслеживают UDP-потоки в сторону IP-адресов Cloudflare. При обнаружении пакета размером 148 байт с характерным заголовком WireGuard соединение блокируется на фазе рукопожатия.

## 🔎 Принцип работы сканеров IP-адресов Cloudflare (WARP-scanner)

Поскольку основные IP-адреса эндпоинтов Cloudflare (такие как `engage.cloudflareclient.com` / `162.159.192.1`) заблокированы ТСПУ на транспортном уровне, для подключения необходимо найти рабочие IP-адреса из пулов Cloudflare.

Для этого используются специализированные сканеры:
*   **Диапазоны сканирования:** Сканируются пулы адресов Cloudflare (например, `162.159.192.0/24`, `162.159.193.0/24`, `188.114.96.0/24`).
*   **Механизм проверки (Probing):** Сканер генерирует пустые UDP-пакеты (или пакеты рукопожатия) и отправляет их на сканируемые IP-адреса на порты `2408`, `500`, `1701`, `4500`.
*   **Измерение RTT и потерь:** Сканер ожидает ответа от серверов Cloudflare. Измеряется время прохождения пакета (RTT) и количество потерянных пакетов (packet loss).
*   **Выбор эндпоинтов:** Рабочими считаются те IP-адреса, которые ответили на запросы (что доказывает отсутствие блокировки на ТСПУ данного провайдера) и показали минимальную задержку. Эти IP-адреса прописываются в поле `Endpoint` файла конфигурации клиента.

## 🚀 Пошаговая инструкция

### Шаг 1. Генерация конфигурации WARP
Для генерации конфига воспользуйтесь готовыми инструментами:
*   **Скрипт в браузере (Google Colab):** Найдите в сети проекты по запросу "Warp-to-WireGuard generator" (например, от *Viacheslav* или аналогичные).
*   **Консольная утилита:** Используйте [wgcf](https://github.com/ViRb3/wgcf):
    ```bash
    wgcf register
    wgcf generate
    ```
    В результате вы получите файл `wgcf-profile.conf`, содержащий ваши приватные ключи (`PrivateKey`), адреса (`Address`) и стандартный заблокированный эндпоинт `engage.cloudflareclient.com:2408`.

---

### Шаг 2. Конвертация в AmneziaWG и настройка обхода DPI
Откройте полученный `.conf` файл в текстовом редакторе и приведите его к формату AmneziaWG.

#### 1. Добавьте параметры обхода в секцию `[Interface]`
Добавьте в секцию `[Interface]` следующие параметры:
```ini
Jc = 4
Jmin = 50
Jmax = 1000
S1 = 15
S2 = 44
H1 = 1
H2 = 2
H3 = 3
H4 = 4
```
*   `Jc` — количество мусорных пакетов при инициализации.
*   `Jmin` и `Jmax` — минимальный и максимальный размер мусора.
*   `S1`, `S2` — смещение размеров пакетов инициализации.
*   `H1`–`H4` — измененные байты заголовков для обхода сигнатурного детекта DPI.

#### 2. Замените заблокированный Endpoint
Стандартный домен `engage.cloudflareclient.com:2408` заблокирован ТСПУ. Вместо него нужно использовать IP-адреса Cloudflare. К сожалению, популярные IP (например, `162.159.192.1`) также могут быть частично заблокированы.

Для поиска работающих IP-адресов используйте сканеры:
*   **[WARP-IP-Scanner (GitHub)](https://github.com/bepass-org/warp-scanner):** Утилита сканирует диапазоны IP Cloudflare и находит те, которые не заблокированы вашим провайдером.
*   Пример рабочего эндпоинта (может меняться): `162.159.193.10:2408` (или порты `500`, `854`, `901`, `1701`, `4500`).

#### Пример итогового конфига `warp_amnezia.conf`:
```ini
[Interface]
PrivateKey = <ВАШ_ПРИВАТНЫЙ_КЛЮЧ>
Address = 172.16.0.2/32, 2606:4700:110:8f8a:e1f5:1234:5678:90ab/128
DNS = 1.1.1.1, 8.8.8.8
# Параметры AmneziaWG:
Jc = 4
Jmin = 50
Jmax = 1000
S1 = 15
S2 = 44
H1 = 1
H2 = 2
H3 = 3
H4 = 4

[Peer]
PublicKey = bmXOC+F1KF...
Endpoint = 162.159.193.5:1701
AllowedIPs = 0.0.0.0/0, ::/0
```

## ⚙️ Оптимизация MTU и маршрутизация

### 1. Специфика MTU для WARP
Размер полезной нагрузки кадра (MTU) для WARP по умолчанию составляет `1420` байт. Однако в мобильных сетях РФ из-за внутренней инкапсуляции оператора (GTP-туннели) эффективный размер кадра часто снижен.
*   При инкапсуляции трафика в AmneziaWG (UDP) реальный размер пакета на физическом интерфейсе возрастает. Если размер превышает MTU мобильного оператора, пакет фрагментируется.
*   ТСПУ сотовых сетей часто отбрасывают фрагментированные UDP-пакеты, что приводит к зависанию туннеля (трафик перестает ходить через несколько секунд после подключения).
*   **Решение:** Установите значение `MTU = 1280` (минимально допустимый размер кадра для IPv6) в секции `[Interface]` конфигурационного файла. Это гарантирует доставку пакетов без фрагментации на любых сотовых сетях.

### 2. Настройка AllowedIPs (Split Tunneling)
Параметр `AllowedIPs` определяет, какой трафик направлять в туннель:
*   **Полный туннель (`AllowedIPs = 0.0.0.0/0, ::/0`):** Весь сетевой трафик устройства шифруется и идет через сеть Cloudflare.
*   **Split Tunneling (Раздельное туннелирование):** Позволяет пустить через WARP только заблокированные ресурсы, а остальной трафик направить напрямую (direct).
    *   *Настройка:* Вы можете указать конкретные подсети заблокированных ресурсов в параметре `AllowedIPs`, чтобы отечественные сервисы (банки, Госуслуги) продолжали работать напрямую с вашим реальным IP.
    *   В мобильных клиентах AmneziaWG разделение трафика удобнее настраивать через встроенную функцию «Исключенные приложения».
